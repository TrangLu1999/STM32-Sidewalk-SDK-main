# Amazon Sidewalk SBDT Demo Application

This application demonstrates how to perform a Firmware Update Over-the-Air (FUOTA) on an STMicroelectronics device using the **Amazon Sidewalk Bulk Data Transfer (SBDT)** feature.

The application connects to the Sidewalk network, downloads a new firmware image via SBDT, and places it into the device's staging area. The update is then securely installed by the Application Install Manager (AIM) bootloader on the next reset.

## Prerequisite: Application Install Manager (AIM)

This demo application **cannot run by itself**. It is designed to be managed by a secure bootloader.

You **must** flash the [sid_application_install_manager](/apps/st/stm32wba/sid_application_install_manager/) project onto your device *before* flashing this application. The AIM acts as the secure bootloader, responsible for verifying the signature of this application before launching it and for handling the installation of new firmware updates.

## How it Works

1.  **Boot:** The AIM bootloader verifies the digital signature of this `sid_sbdt_demo` application and, if valid, launches it.
2.  **Download:** The application runs and uses the Sidewalk stack to initiate an SBDT (Sidewalk Bulk Data Transfer) process.
3.  **Receive Update:** It receives the new firmware update image from the Sidewalk network.
4.  **Stage Firmware:** The downloaded image is written to the "Staging Area" in the device's flash memory.
5.  **Reboot & Install:** When the download is complete, the application triggers a system reset. The AIM bootloader takes control, detects the new valid image in the staging area, verifies its signature, and securely installs it, completing the FUOTA process.

## Build and Signing Process

This project is configured for a seamless, secure build process right out of the box.

### Automatic Signing

You do not need to perform any manual signing steps.

This STM32CubeIDE project is configured with a **post-build step** that automatically executes the [Firmware Signing Tool](/tools/firmware_signing/) every time you build the project. This ensures that your firmware is always correctly signed for the AIM bootloader.

### Build Outputs

The automatic post-build step generates two sets of files:

1.  **Signed Active Application (e.g., `sid_sbdt_demo_wba65.elf`)**
    * This is the main application binary, signed to be a valid, bootable "active" application.
    * **Use this file for the initial flashing** of the application *after* you have flashed the AIM bootloader.

2.  **OTA Update Image (e.g., `sid_sbdt_demo_wba65_ota_image.bin`)**
    * This is the firmware package intended for over-the-air delivery.
    * **Use this file for the FUOTA process.** Upload this file to your S3 bucket to be delivered via the Amazon Sidewalk SBDT service.

## Flashing and Usage

1.  **Erase** the device completely.
2.  **Flash the Bootloader:** First, flash the `sid_application_install_manager` binary to the device.
3.  **Flash the Manfucaturing Data:** Second, flash the device credentials.
4.  **Flash the Application:** Use STM32CubeIDE build to run or debug this application.
5.  The device will now boot: AIM will verify `sid_sbdt_demo` and then launch it.
6.  **Perform an OTA Update:**
    * Take the `_ota_image.bin` file generated by a build (e.g., from a newer version of the app).
    * Upload this file to your S3 bucket configured for Sidewalk SBDT.
    * Initiate the FUOTA process from your AWS account.
    * The device will download and install the update, demonstrating the complete FUOTA loop.
